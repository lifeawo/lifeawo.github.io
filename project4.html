Third Step:

Linking Constraints to Cells

As a final step, the former two need to be linked. The machine must attribute the values of the Cahier des Charges to the novel's chapters.

Every chapter has a value for every one of the 42 categories, obtained by applying the Double Latin Square and the "pseudo-quenine" base 10. This attribution never changes, even if we change the constraints, their values, or the places and characters of the story.

This story-making machine works for other novels based on a 10x10 map with 42 categories of constraints.

CQ: What is the value of constraint “époque” for chapter 8?

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX lawo: <https://lifeawo.github.io/lifeawo.owl#>
PREFIX bag: <http://www.ontologydesignpatterns.org/cp/owl/bag.owl#>

SELECT ?constvaluelabel
  WHERE {
    ?const rdfs:label "époque" .
    ?constvalue bag:itemOf ?const .
    ?chap lawo:hasConstraintValue ?constvalue ;
      lawo:ChapterNumber 8 .
    ?constvalue rdfs:label ?constvaluelabel .
  }

Result:
constvaluelabel = "18ème"

CQ: What are the constraints of the missing cell/chapter?

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX lawo: <https://lifeawo.github.io/lifeawo.owl#>
PREFIX bag: <http://www.ontologydesignpatterns.org/cp/owl/bag.owl#>

SELECT ?constr ?constrlabel ?constrvallabel (count(?chap) as ?cnt)
  WHERE {
    ?constrval a lawo:ConstraintValue ;
               lawo:belongsToChapter ?chap ;
               rdfs:label ?constrvallabel ;
               bag:itemOf ?constr .
    ?constr rdfs:label ?constrlabel .
  }
GROUP BY ?constr ?constrlabel ?constrvallabel
HAVING (?cnt<10)
ORDER BY ?cnt ?constr ?constrval
